电商首单欺诈检测：召回天花板与阈值假象

业务挑战

1.02% 的极端不平衡（1,415 / 138,376）让准确率失效——全判"正常"也有 98.98% 准确率却抓到零欺诈。目标是在不伤害新客的约束下最大化欺诈捕获。

解决方案

四个模型走同一套无泄漏协议公平对比，PR-AUC 全部打平在 0.535–0.541；选 LightGBM+SMOTE，并做成本敏感的阈值决策。

商业价值

拦截 0.53% 流量抓住 51.6% 欺诈、仅 2 个误报；金额加权欺诈损失 $14,503 → $6,939，降低约 52%。

00 Executive Summary

执行摘要

对 138,376 笔电商首单交易的分析，回答三个问题：用新用户的开局购买能多可靠地预测欺诈，为什么无论换什么模型召回都卡在 52% 左右，以及为什么默认阈值下的 98.6% 精确率衡量的是阈值而非模型。最终模型是 LightGBM + SMOTE，按 PR-AUC 调参。

四个发现

  • 信号集中在两个特征，其余都是噪声。注册→购买时间间隔和设备共享账号数几乎承担全部判别力（SHAP 均值 2.52 与 2.42，其余均低于 0.7）；年龄、性别、金额、渠道、国家都没有可用信号。
  • 召回被数据锁死在约 52%，不是模型问题。半数欺诈——注册后 1 秒内下单的 739 个账号——完全可分；另一半与正常首单几乎无法区分，因此召回稳定在 52% 左右，不随阈值或模型变化。
  • 固定阈值下的精确率反映分数量纲，不是模型质量。同一 LightGBM 换成类别权重，排序能力完全相同（PR-AUC 0.54），0.5 阈值下精确率却只有 34%。
  • 大多数类别信号过不了显著性检验。营销渠道不显著（卡方 p=0.27），周末注册不显著（p=0.26），国家仅凭样本量达到显著、幅度只有 0.44%–1.91%。

01 The Data

数据与不平衡

数据很干净：138,376 行，无缺失、无重复。唯一主导性的事实是不平衡——欺诈率 1.02%，约 1/97。而只有 1,415 个欺诈样本，模型只能抓住强且重复的信号，这正是它最终只依赖两个特征的原因。

Method · 为什么评估用 PR-AUC，IP 用区间连接

准确率失效。对所有人预测"正常"就能拿 98.98% 的准确率却抓到零欺诈，所以评估改用精确率、召回率和 PR-AUC——后者不同于 ROC-AUC，对稀有类敏感。

IP→国家的高效还原。逐行扫描查找表是 O(n·m)；把两张表排序后用 merge_asof 做区间连接可一次完成。14.7% 的 IP 落在所有区间之外，标为 Unknown。

02 Exploratory Analysis

探索性分析说明了什么

购买速度是主导信号。正常用户从注册到首单的中位间隔约 60 天，欺诈的中位间隔是 1 秒。1 秒内完成的 739 笔交易 100% 是欺诈，占全部欺诈的 52%。另一半欺诈散布在看似正常的间隔里——这是召回天花板的第一个迹象。

设备共享是第二个信号，而金额根本不是信号。只绑定一个账号的设备几乎从不欺诈（0.35%）；一旦三个及以上账号共用一台设备，欺诈率升到 58%。金额则在两类间几乎相同（欺诈均值 $37.9 vs 正常 $36.9）——欺诈者刻意在普通金额区间交易。

Significance · 渠道和浏览器的故事站不住

按欺诈率排序时差异看着有戏，显著性检验却把它们抹平：渠道不显著（p=0.27），浏览器仅凭样本量达到 p=0.007、幅度只有可忽略的 0.86%–1.13%。只有 1,415 个欺诈样本时，微小的比率差异极易被过度解读，所以每个差异都先检验再采信。

03 Model Bake-off

公平比较的模型

三个模型家族走同一套无泄漏流程：80/20 分层划分，设备与 IP 的共享计数用只在训练折上拟合的 CountEncoder 编码，SMOTE 放进流水线以确保合成样本绝不泄漏进验证折，5 折 CV 用 average precision 评分。第四个变体——用 scale_pos_weight、不做重采样的 LightGBM——用来检验 SMOTE 是否改变排序。

模型 PR-AUC ROC-AUC
Logistic Regression (SMOTE) 0.535 0.757
Random Forest (SMOTE) 0.541 0.762
LightGBM (SMOTE) 0.540 0.758
LightGBM (weighted) 0.541 0.763

四者都落在 0.535–0.541 的区间，其宽度小于 ±0.05 的折间标准差——它们打平。我们选用调参后的 LightGBM + SMOTE（测试集 PR-AUC 0.535），同时指出逻辑回归本可作为更简单、同样成立的选择。

Artifact · 98.6% 精确率衡量的是阈值，不是模型

模型在 0.5 阈值下达到 98.6% 精确率，但同一 LightGBM 换成类别权重（scale_pos_weight）而非 SMOTE，排序能力完全相同（PR-AUC 0.54），在 0.5 阈值下精确率却只有 34%——因为它把分数放在了不同的量纲上（预测概率均值 0.125，对比 SMOTE 版的 0.010）。固定阈值下的精确率反映的是阈值落在双峰分数分布的哪个位置，而不是哪个模型更好。

04 The Recall Ceiling

召回天花板

精确率-召回率曲线在召回 0.52 之前维持精确率约 1.0，随后断崖下跌。扫描阈值可见召回全程平贴在 0.52，而精确率在爬升——阈值换来的是精确率，不是召回。分数直方图解释了原因：预测值堆在 0 附近或 1 附近，中间几乎没有。接近 1.0 的欺诈是 instant-purchase 那一半，唾手可得；另一半的分数低到与正常流量无异，任何阈值都救不回来。

这是特征集的诚实上限。测试集上调参模型抓住 283 笔欺诈中的 146 笔（51.6%），误报为 27,393 笔正常交易中的 2 笔（混淆矩阵 TN 27,391 / FP 2 / FN 137 / TP 146）。

Data caveat · 线上召回大概率更低

本数据集的构造让 instant-purchase 与设备共享模式异常干净，这正是前两个特征如此主导、且 instant-purchase 簇 100% 欺诈的原因。真实流量里这些信号会更嘈杂，所以线上召回大概率低于 52%，而非更高。

05 Cost & Threshold

成本与阈值决策

0.5 阈值不是业务决策。给两类错误定价能把权衡摆明：漏报的成本是丢失的商品加一笔固定拒付费，按每笔交易的真实金额计算；误报的成本是拦下一位优质新客的固定罚金，其真实价值是业务假设而非可测数字，故做敏感性扫描。

误报罚金 最优阈值 总成本 对比全放行
$50 0.64 $6,939 −52%
$100 0.64 $6,939 −52%
$200 0.64 $6,939 −52%

因为分数是双峰的，最优阈值对罚金的具体取值不敏感——$50 到 $200 都落在 0.6 附近，金额加权的欺诈损失（$14,503 → $6,939）比全放行低约 52%（模型抓住了 instant-purchase 那一半，它承担了约一半的总欺诈金额）。极高的误报成本会把阈值推高到纯 instant-purchase 簇（召回约 0.52，几乎零误报）；较低的误报成本会把它拉低，用精确率换几个额外的抓获。平坦的最优区间让业务可以优先保精确率而几乎不损召回。

06 What the Model Uses

模型依赖什么

测试集上的 SHAP 用平实的话印证了两特征的故事：signup 到 purchase 的短间隔是最强的"欺诈"推力（真实客户会浏览、会再来；自动化或盗用凭证的购买在几秒内完成），设备被多账号共享是第二强（欺诈团伙的指纹）。其余一切——性别、周末、浏览器、小时——只贡献前两者权重的一小部分，更应读作轻微相关而非真实驱动。模型忽略了年龄、金额和营销渠道。

07 Productionizing

把分数落成产品决策

对最终模型测试集分数分箱，能看出真实的分层长什么样：

层级 流量占比 欺诈率 抓获欺诈
高（≥ 0.7）：拒绝 + 人工复核 0.53% 100% 146
中（0.3–0.7）：二次验证 0.01% 0% 0
低（< 0.3）：自动放行 99.5% 0.5% 137
Insight · 中间层几乎为空

高风险层——0.53% 流量、几乎全是欺诈——实时拒绝并转人工复核，抓住约 52% 的欺诈且几乎零误报。低风险层无摩擦自动放行，也正是被漏掉的那一半欺诈藏身之处。中间层几乎为空：模型不会在 0.5–0.9 产生有意义的"不确定"人群，所以基于这些分数搭的二次验证通道会几乎空转。要多抓欺诈，就得主动把高风险层阈值下调进低精确率区间，每多抓一笔都要付出迅速上升的误报。

上线前检查

  • 用按时间排序的回测验证共享计数特征（它们目前假设了全量历史）。
  • 在线上流量对照半合成的干净划分监控精确率与召回。
  • 先在小流量切片上做 A/B 测试，跟踪抓获欺诈、误报率和首单转化。
