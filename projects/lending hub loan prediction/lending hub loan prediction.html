00 Executive Summary

执行摘要

业务挑战：贷款组合面临严重的16.8%违约率，导致1.1亿美元组合中1840万美元的损失。核心挑战是在不拒绝过多信誉良好申请人而影响业务增长的情况下，显著减少这些信贷损失。错过违约（假阴性，平均13,663美元）的高财务成本远超拒绝良好贷款（假阳性，约2,160美元）的机会成本，需要一个优化成本而非仅仅准确性的模型。

解决方案：我们开发了一个强大的端到端机器学习系统，核心是成本优化的随机森林模型。通过将决策阈值从默认的0.5调整到最优的0.19，该模型被精确调优以最小化预测错误的总财务成本。这个预测模型配合LightGBM模型的深度SHAP分析，将复杂的"黑箱"决策转化为透明、可操作的业务规则和详细的风险画像。

关键风险因素的违约率分析 - G级显示52%违约率 vs A级7%，60个月期隄32%违约 vs 36个月期隄11%

风险因素分析洞察

关键风险指标

1. 信用等级 - 指数级风险增长
  • G级显示灾难性违约率（52%）- 超过A级（7%）的7倍
  • 明显的指数模式：每个等级降级增加约40%的风险
  • E-G级仅占贷款的10%但导致25%的违约

业务行动：在E级实施硬性截止或对F-G级应用3倍定价溢价

2. 贷款期限 - 期限风险溢价
  • 60个月贷款违约率（32%）几乎是36个月（11%）的3倍
  • 延长期限增加21个百分点的违约概率
  • 更长期限 = 更高不确定性 + 支付疲劳

业务行动：为60个月期限增加5-7%的利息溢价或仅限于A-B级

风险分层框架

基于分析，实施四层风险系统：

Prime Tier
  • 特征：A-B 级，36 个月，有房产 / 自有
  • 违约率：<10%
  • 行动：自动批准，最优利率
Standard Tier
  • 特征：C-D 级，任何期限，已验证
  • 违约率：15–20%
  • 行动：标准流程，市场利率
Sub-Prime Tier
  • 特征：E 级，60 个月，租房
  • 违约率：25–35%
  • 行动：增强审查，溢价利率
High Risk Tier
  • 特征：F-G 级，小企业贷款
  • 违约率：>40%
  • 行动：人工审查或拒绝

红旗组合

最高风险画像（>40%违约概率）：

  1. F-G级 + 60个月期限
  2. E级 + 小企业目的
  3. <1年就业 + 高DTI + D级或更差

分布比较：全额支付vs冲销贷款 - 关键特征清晰分离

特征相关矩阵及与违约相关的前15个特征

地理风险分析 - 阵拉斯加州、南达科他州和内华达州风险较高

客户细分分析 - 优质客户群vs深度次级客户

反直觉的发现

  1. 收入悖论：验证收入显示更高的违约率
    • 假设：边缘申请人需要验证
    • 行动：使用验证原因作为额外特征
  2. 小额贷款神话：较大贷款违约更频繁
    • 传统假设：小额贷款=绝望的借款人
    • 现实：大额贷款=过度自信的借款人
    • 行动：对25,000美元以上贷款加强控制
  3. 地理惊喜：内华达州（27%违约）vs犹他州（12%）
    • 文化和经济因素很重要
    • 行动：纳入州经济指标

步骤1：处理缺失值

插补的业务逻辑：

  • emp_length：316个缺失 → 使用0（新员工或自雇）
  • revol_util：3个缺失 → 使用中位数（典型利用率）
# 使用正确的插补方法处理缺失值
LC_df_fe['emp_length'] = LC_df_fe['emp_length'].fillna('< 1 year')
LC_df_fe['revol_util'] = LC_df_fe['revol_util'].fillna(LC_df_fe['revol_util'].median())

# 将earliest_cr_line转换为信贷历史年数
def calculate_credit_history(excel_date):
    try:
        # Excel日期：自1900-01-01以来的天数
        base = datetime(1900, 1, 1)
        date = base + timedelta(days=float(excel_date)-2)
        return (datetime.now() - date).days / 365.25
    except:
        return LC_df_fe['credit_history'].median()

LC_df_fe['credit_history'] = LC_df_fe['earliest_cr_line'].apply(calculate_credit_history)

特征工程策略

基于业务理解和EDA洞察，创建了多个复合风险特征来捕获贷款违约的复杂模式。

核心工程特征

  • 财务压力指数：综合DTI、信贷利用率和查询次数
  • 风险调整利率：相对于同等级平均利率的偏差
  • 贷款负担比率：年还款额占年收入比例
  • 等级-期限风险：捕获等级与期限的交互效应
  • 信贷利用率分层：低（<30%）、中（30-70%）、高（>70%）
特征工程实现： 
# 创建复合风险特征
LC_df['financial_stress_index'] = (
    (LC_df['dti'] / 100) * 
    (LC_df['revol_util'] / 100) * 
    LC_df['inq_last_6mths']
) / LC_df['annual_inc']

# 风险调整利率
LC_df['risk_adjusted_rate'] = (
    LC_df['int_rate'] - 
    LC_df.groupby('grade')['int_rate'].transform('mean')
)

# 贷款负担比率
LC_df['loan_burden_ratio'] = (
    (LC_df['installment'] * 12) / LC_df['annual_inc']
)

# 信贷利用率分层
LC_df['moderate_credit_util'] = (
    (LC_df['revol_util'] >= 30) & 
    (LC_df['revol_util'] <= 70)
).astype(int)

类别不平衡处理：SMOTE策略

原始数据集中违约贷款仅占15.8%，通过SMOTE技术提升少数类样本，改善模型对违约贷款的识别能力。

SMOTE参数设置

  • 原始：15.8%违约率（1:5.3比例）
  • 目标：SMOTE后33%（1:2比例）

def find_optimal_threshold(y_true, y_proba, min_recall=0.75):
    thresholds = np.arange(0.05, 0.95, 0.01)
    best_threshold = 0.5
    min_cost = float('inf')
    
    for threshold in thresholds:
        y_pred = (y_proba >= threshold).astype(int)
        cm = confusion_matrix(y_true, y_pred)
        tn, fp, fn, tp = cm.ravel()
        
        # 业务成本计算
        fn_cost = fn * 13663  # 错过违约
        fp_cost = fp * 2160   # 错过机会
        total_cost = fn_cost + fp_cost
        
        recall = tp / (tp + fn) if (tp + fn) > 0 else 0
        if recall >= min_recall and total_cost < min_cost:
            min_cost = total_cost
            best_threshold = threshold
    
    return best_threshold

    verbose=-1
)

# 拟合模型
lgb_model.fit(X_train_balanced_scaled, y_train_balanced)

# 为SHAP分析创建DataFrame（带特征名）
feature_names = X.columns.tolist()
X_test_scaled_df = pd.DataFrame(X_test_scaled, columns=feature_names)
X_train_balanced_scaled_df = pd.DataFrame(X_train_balanced_scaled, columns=feature_names)

# 创建SHAP解释器
explainer = shap.TreeExplainer(
    lgb_model,
    X_train_balanced_scaled_df,
    feature_perturbation='interventional',
    model_output='raw'
)

# 计算SHAP值
shap_values = explainer.shap_values(X_test_scaled_df, check_additivity=False)

# 计算特征重要性（平均绝对SHAP值）
mean_abs_shap = np.abs(shap_values).mean(axis=0)
shap_importance = pd.DataFrame({
    'feature': feature_names,
    'mean_abs_shap': mean_abs_shap,
    'mean_shap': shap_values.mean(axis=0),  # 影响方向
    'std_shap': shap_values.std(axis=0)
}).sort_values('mean_abs_shap', ascending=False)

print("前10个最重要特征:")
print(shap_importance.head(10))

# 生成SHAP可视化
# 1. 特征重要性图
shap.summary_plot(shap_values, X_test_scaled_df, feature_names=feature_names, 
                  plot_type="bar", show=False)
plt.title('SHAP特征重要性')
plt.tight_layout()
plt.show()

# 2. 摘要图（蜂群图）
shap.summary_plot(shap_values, X_test_scaled_df, feature_names=feature_names, show=False)

plt.show()

非线性关系

  • 低利用率（<30%）：轻微风险增加 - 可能是不活跃卡片或信贷经验不足

  • 高利用率（>70%）：风险急剧增加 - 财务压力信号

这一发现挑战了传统智慧，建议采用更细致的承保方法。
