00 Executive Summary

Executive Summary

Business Challenge: The loan portfolio faces a severe 16.8% default rate, resulting in $18.4M losses on a $110M portfolio. The core challenge is significantly reducing these credit losses without rejecting too many creditworthy applicants, which would hamper business growth. The high financial cost of missed defaults (false negatives, averaging $13,663) far exceeds the opportunity cost of rejecting good loans (false positives, approximately $2,160), necessitating a model optimized for cost rather than just accuracy.

Solution: We developed a robust end-to-end machine learning system centered on a cost-optimized Random Forest model. By adjusting the decision threshold from the default 0.5 to an optimal 0.19, the model is precisely tuned to minimize the total financial cost of prediction errors. This predictive model, coupled with deep SHAP analysis from a LightGBM model, transforms complex "black-box" decisions into transparent, actionable business rules and detailed risk profiles.

def calculate_credit_history(excel_date):
    try:
        # Excel date: days since 1900-01-01
        base = datetime(1900, 1, 1)
        date = base + timedelta(days=float(excel_date)-2)
        return (datetime.now() - date).days / 365.25
    except:
        return LC_df_fe['credit_history'].median()

LC_df_fe['credit_history'] = LC_df_fe['earliest_cr_line'].apply(calculate_credit_history)

Feature Engineering Strategy

Based on business understanding and EDA insights, we created multiple composite risk features to capture complex patterns of loan default.

Core Engineered Features

  • Financial Stress Index: Combines DTI, credit utilization, and inquiries
  • Risk-Adjusted Rate: Deviation from average rate for same grade
  • Loan Burden Ratio: Annual payments as percentage of income
  • Grade-Term Risk: Captures interaction between grade and term
  • Credit Utilization Tiers: Low (<30%), Moderate (30-70%), High (>70%)
Feature Engineering Implementation: 
# Create composite risk features
LC_df['financial_stress_index'] = (
    (LC_df['dti'] / 100) * 
    (LC_df['revol_util'] / 100) * 
    LC_df['inq_last_6mths']
) / LC_df['annual_inc']

# Risk-adjusted rate
LC_df['risk_adjusted_rate'] = (
    LC_df['int_rate'] - 
    LC_df.groupby('grade')['int_rate'].transform('mean')
)

# Loan burden ratio
LC_df['loan_burden_ratio'] = (
    (LC_df['installment'] * 12) / LC_df['annual_inc']
)

# Credit utilization tiers
LC_df['moderate_credit_util'] = (
    (LC_df['revol_util'] >= 30) & 
    (LC_df['revol_util'] <= 70)
).astype(int)

# Cost-sensitive threshold optimization
def find_optimal_threshold(y_true, y_proba, min_recall=0.75):
    thresholds = np.arange(0.05, 0.95, 0.01)
    best_threshold = 0.5
    min_cost = float('inf')
    
    for threshold in thresholds:
        y_pred = (y_proba >= threshold).astype(int)
        cm = confusion_matrix(y_true, y_pred)
        tn, fp, fn, tp = cm.ravel()
        
        # Business cost calculation
        fn_cost = fn * 13663  # Missed defaults
        fp_cost = fp * 2160   # Missed opportunities
        total_cost = fn_cost + fp_cost
        
        recall = tp / (tp + fn) if (tp + fn) > 0 else 0
        if recall >= min_recall and total_cost < min_cost:
            min_cost = total_cost
            best_threshold = threshold
    
    return best_threshold

# Create SHAP explainer
explainer = shap.TreeExplainer(
    lgb_model,
    X_train_balanced_scaled_df,
    feature_perturbation='interventional',
    model_output='raw'
)
